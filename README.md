## Overview

This project implements a **compact semantic segmentation pipeline** for the PASCAL VOC 2012 dataset. The model (`MobileNetV3LightSeg`) is engineered to minimize FLOPs and parameters while maintaining competitive segmentation accuracy, making it suitable for deployment on resource-constrained devices.

**Key highlights:**
-  Ultra-lightweight backbone: MobileNetV3-Small
-  Strict 300×300 end-to-end inference
-  Binary Dice metric for foreground/background evaluation
-  Robustness testing with corrupted inputs (noise, blur)
-  Supports CUDA, CPU, and Intel DirectML (Windows)

---

## Model Architecture

```
Input (3 × 300 × 300)
        │
   MobileNetV3-Small Backbone (encoder, layers 0–9)
        │
   Skip connections at layers: [3, 8, 9]  →  feature maps at 75×75, 38×38, 10×10
        │
   Global Context Module (AdaptiveAvgPool + Sigmoid attention)
        │
   Reduce Conv (96 → 64 channels)
        │
   Decoder:
     ├── Upsample → Fuse with 38×38 skip (SeparableConv: 64+48 → 32)
     └── Upsample → Fuse with 75×75 skip (SeparableConv: 32+24 → 16)
        │
   Dropout2D (p=0.1)
        │
   Classification Head (16 → num_classes, 1×1 Conv)
        │
   Bilinear Upsample → 300×300
        │
 [Train] Raw logits      [Eval/Inference] argmax → Integer Mask
```

**Components:**
- `SeparableConv2d` — Depthwise separable convolution with BatchNorm + Hardswish
- `GlobalContext` — Channel-wise attention via global average pooling
- `MobileNetV3LightSeg` — Full encoder-decoder segmentation model

## Dataset Setup

This project uses the **PASCAL VOC 2012** train/val dataset.

1. Download the dataset from [official VOC site] or Kaggle[https://www.kaggle.com/datasets/gopalbhattrai/pascal-voc-2012-dataset].
2. Extract it so the directory structure looks like this:

```
VOC2012_train_val/
├── JPEGImages/          ← RGB images (.jpg)
├── SegmentationClass/   ← Segmentation masks (.png)
└── ...
```

3. Only images that have a corresponding mask in `SegmentationClass/` are used — `prepare_data.py` handles this filtering automatically.

## Usage

### 1. Prepare Data

Validates the dataset, creates an 80/20 train/val split, and runs a dataloader sanity check.

```bash
python prepare_data.py \
  --voc_root /path/to/VOC2012_train_val \
  --out_splits voc2012_splits.json \
  --train_frac 0.8 \
  --seed 42 \
  --img_size 300 \
  --batch_size 4
```

| Argument | Default | Description |
|---|---|---|
| `--voc_root` | *Will get after running code* | Path to VOC root directory |
| `--out_splits` | `voc2012_splits.json` | Output path for split JSON |
| `--train_frac` | `0.8` | Fraction of data for training |
| `--seed` | `42` | Random seed for reproducibility |
| `--force_resplit` | `False` | Flag to regenerate splits |

---

### 2. Train

Trains the model with a combined **CrossEntropy + Dice loss**, cosine LR scheduler, and optional AMP. Supports hyperparameter grid search.

```bash
python train.py \
  --voc_root /path/to/VOC2012_train_val \
  --splits_json voc2012_splits.json \
  --output_dir ./checkpoints \
  --epochs 100 \
  --batch_size 8 \
  --lr 1e-3 \
  --img_size 300 \
  --use_amp
```

| Argument | Default | Description |
|---|---|---|
| `--voc_root` | *(Will get after running code)* | Path to VOC root directory |
| `--splits_json` | *(Will get after running code)* | Path to splits JSON |
| `--output_dir` | `./checkpoints` | Directory to save checkpoints |
| `--epochs` | `100` | Total training epochs |
| `--batch_size` | `8` | Training batch size |
| `--lr` | `1e-3` | Initial learning rate |
| `--weight_decay` | `1e-4` | AdamW weight decay |
| `--patience` | `10` | Early stopping patience |
| `--corruption_prob` | `0.3` | Augmentation corruption probability |
| `--use_amp` | `True` | Enable Automatic Mixed Precision |
| `--pretrained` | `True` | Use pretrained MobileNetV3 backbone |
| `--resume` | `""` | Path to checkpoint to resume from |
| `--use_dml` | `False` | Use Intel DirectML (Windows) |
| `--no_cuda` | `False` | Force CPU training |

Checkpoints saved to `output_dir/`:
- `best_checkpoint.pth` — Best validation Dice model
- `last_checkpoint.pth` — Most recent epoch
- `tb/` — TensorBoard logs

**Monitor training with TensorBoard:**
```bash
tensorboard --logdir ./checkpoints/tb
```

### 3. Evaluate

Runs three evaluation tests: **clean accuracy**, **corrupted robustness**, and **computational efficiency (FLOPs)**.

```bash
python eval.py \
  --voc_root /path/to/VOC2012_train_val \
  --splits_json voc2012_splits.json \
  --checkpoint_path ./checkpoints/best_checkpoint.pth \
  --batch_size 8
```

| Argument | Default | Description |
|---|---|---|
| `--voc_root` | *(Will get after running code)* | Path to VOC root directory |
| `--splits_json` | *(Will get after running code)* | Path to splits JSON |
| `--checkpoint_path` | *(Will get after running code)* | Path to `.pth` model checkpoint |
| `--batch_size` | `8` | Evaluation batch size |
| `--img_size` | `300` | Input image resolution |
| `--num_classes` | `21` | Number of segmentation classes |
| `--use_dml` | `False` | Use Intel DirectML (Windows) |

**Sample output:**

### 4. Inference

Runs the model on a folder of test images and saves **binary masks** (black = background, white = foreground) to the output directory.

```bash
python inference.py \
  --in_dir /path/to/test/images \
  --out_dir group_15_output \
  --checkpoint_path ./checkpoints/best_checkpoint.pth \
  --img_size 300
```

| Argument | Default | Description |
|---|---|---|
| `--in_dir` | *(Will get after running code)* | Path to input images folder |
| `--out_dir` | `group_15_output` | Folder to save output binary masks |
| `--checkpoint_path` | *(Will get after running code)* | Path to `.pth` model checkpoint |
| `--img_size` | `300` | Input resolution (must be 300) |
| `--use_dml` | `False` | Use Intel DirectML (Windows) |

> ⚠️ The output folder **must** be named `group_15_output` for competition submission.

## Training Details

| Component | Details |
|---|---|
| **Backbone** | MobileNetV3-Small (pretrained on ImageNet) |
| **Loss Function** | CrossEntropyLoss (class-weighted) + Dice Loss |
| **Optimizer** | AdamW (lr=1e-3, weight decay=1e-4) |
| **LR Scheduler** | CosineAnnealingLR |
| **Input Size** | 300 × 300 |
| **Augmentations** | Flip, Affine, ColorJitter, GaussNoise, MotionBlur, CoarseDropout |
| **AMP** | Enabled (CUDA / DirectML) |
| **Early Stopping** | Patience = 10 epochs |

---

## Evaluation Metrics

- **Binary Dice Score** — Primary metric. Measures overlap between predicted and ground truth foreground regions.

```
Dice = (2 × |Pred ∩ GT|) / (|Pred| + |GT|)
```

- **Macro Dice** — Average Dice across all 21 VOC classes (used during training validation).
- **FLOPs** — Measured via the `thop` library for a single 300×300 input.

## Notes

- On **Windows**, set `--num_workers 0` to avoid multiprocessing issues with DataLoader.
- The `KMP_DUPLICATE_LIB_OK=TRUE` environment variable is set automatically in `train.py` and `prepare_data.py` to prevent OpenMP conflicts.
- The model operates in **two modes**:
  - `model.train()` → returns raw logits `(B, C, H, W)` for loss computation.
  - `model.eval()` → returns integer class masks `(B, H, W)` directly.
- `voc2012_splits.json` is generated by `prepare_data.py`. The version in this repo contains empty lists and must be regenerated with your local dataset path.
